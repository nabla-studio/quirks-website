The library currently consists of 7 different packages:

<Files>
  <File title="@quirks/" defaultOpen>
    <File title="core">
      <Callout>
        defines all logic and data structures of quirks, such as classes for
        wallets.
      </Callout>
    </File>
    <File title="store">
      <Callout>
        implement core logic using zustand to have a responsive and global state
        manager.
      </Callout>
    </File>
    <File title="wallets">
      <Callout>
        contains the implementation of providers for both web and mobile
        wallets.
      </Callout>
    </File>
    <File title="react">
      <Callout>
        exposes the store using react hooks and providing utilities.
      </Callout>
    </File>
    <File title="react-native">
      <Callout>exposes config and utilities for react native.</Callout>
    </File>
    <File title="vue">
      <Callout>
        exposes the vue using react hooks and providing utilities.
      </Callout>
    </File>
    <File title="nuxt">
      <Callout>
        implements vue hooks with autocomplete and ssr management for nuxt.
      </Callout>
    </File>
  </File>
  <File title="@nabla-studio/" defaultOpen>
    <File title="chain-registry">
      <Callout>
        contains all the information of the chains and their assets as defined
        within the cosmos chain registry, unlike other packages we do not export
        all the information into one object but the elements are split so as to
        enable three-shake and avoid importing data not needed by the app but
        only what is required.
      </Callout>
    </File>
    <File title="wallet-registry">
      <Callout>
        contains all the information of the wallets as defined within the cosmos
        wallet registry, unlike other packages we do not export all the
        information into one object but the elements are split so as to enable
        three-shake and avoid importing data not needed by the app but only what
        is required.
      </Callout>
    </File>
  </File>
</Files>
